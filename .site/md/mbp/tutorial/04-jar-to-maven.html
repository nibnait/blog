    # 发布自己项目 JAR 到 MAVEN 中央仓库

    我的 issues: https://issues.sonatype.org/browse/OSSRH-79805?filter=-2 (opens new window)

    # 1. 注册 sonatype 的 Jira 帐号

    Sonatype通过JIRA来管理OSSRH仓库。JIRA是一个项目管理服务，类似于国内的Teambition。 注册地址: https://issues.sonatype.org/secure/Signup!default.jspa (opens new window)

    # 项目的发布申请

    # （按提示来就好了）

    https://issues.sonatype.org/projects/OSSRH/issues/OSSRH-79805?filter=allissues (opens new window)

    https://central.sonatype.org/publish/publish-guide/#deployment (opens new window) https://central.sonatype.org/publish/release/ (opens new window)

    # 2. 下载并安装GPG

    下载并安装GPG Suite https://gpgtools.org/ (opens new window)

    # 3. 修改 setting.xml 和 pom.xml

    # setting.xml

    一定要修改 ~/.m2/setting.xml，其他路径、其他文件名 都不行

    新增一个 server

          <server>
        <id>sonatype-nexus-snapshots</id>
        <username>nibnait</username>
        <password>xxx</password>
      </server>

username 和 password 是上面 注册 sonatype 时 填的账号密码

    # pom.xml

    // 第一次上传 一定要 是 release 版本
<version>1.0-RELEASE</version>

<name>common-enhance</name>
<url>https://github.com/nibnait/common-enhance</url>
<description>
  java common util collection
</description>

    下面 这一堆 直接丢到 下面 就好了

        <licenses>
        <license>
            <name>BSD 3-Clause</name>

        </license>
    </licenses>
    <scm>
        <connection>https://github.com/nibnait/common-enhance.git</connection>
        <url>https://github.com/nibnait/common-enhance</url>
    </scm>
    <developers>
        <developer>
            <name>nibnait</name>
            <email>nibnait@icloud.com</email>
            <roles>
                <role>Developer</role>
            </roles>
            <timezone>+8</timezone>

    </developers>

    <profiles>
        <profile>
            <id>release</id>
            <build>

                </resources>

                                <goals>
                                    <goal>jar-no-fork</goal>
                                </goals>
                            </execution>
                        </executions>
                    </plugin>
                    <!-- Javadoc -->
                    <plugin>
                        <groupId>org.apache.maven.plugins</groupId>
                        <artifactId>maven-javadoc-plugin</artifactId>

                            <show>private</show>

                            <!-- TODO 临时解决不规范的javadoc生成报错,后面要规范化后把这行去掉 -->

                        <groupId>org.apache.maven.plugins</groupId>
                        <artifactId>maven-gpg-plugin</artifactId>

                        <extensions>true</extensions>
                        <configuration>
                            <serverId>sonatype-nexus-snapshots</serverId>
                            <nexusUrl>https://s01.oss.sonatype.org/</nexusUrl>

                    </plugin>

                </plugins>
            </build>
            <distributionManagement>
                <snapshotRepository>
                    <id>sonatype-nexus-snapshots</id>
                    <name>Sonatype Nexus Snapshots</name>
                    <url>https://s01.oss.sonatype.org/content/repositories/snapshots/</url>
                </snapshotRepository>
                <repository>
                    <id>sonatype-nexus-snapshots</id>
                    <name>Nexus Release Repository</name>
                    <url>https://s01.oss.sonatype.org/service/local/staging/deploy/maven2/</url>
                </repository>
            </distributionManagement>
        </profile>
    </profiles>

    这里遇到的坑： https://developer.aliyun.com/article/844214 (opens new window) https://blogtech.top/2019/12/30/发布自己的项目 JAR 到 MAVEN 中央仓库 (opens new window) 参考了两篇文章，（但是 可能是因为文章有点老），上面的 distributionManagement -> repository 配置的 都是 https://oss.sonatype.org。我这边照着配置了之后，一直报401 Unauthorized。

    后来 仔细读了 issues (opens new window) 里的 英格力是 才发现，21年2后以后，repository 地址改到 https://s01.oss.sonatype.org 了👇 。。然后又通篇读了遍教程，改了改其他配置，才成功上传 https://central.sonatype.org/publish/publish-guide/#deployment (opens new window)
